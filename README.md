Project Description

This task focuses on database schema design and data integrity using SQL. It demonstrates:

Choosing correct SQL data types

Applying constraints for validation

Modifying table structure using ALTER TABLE

Understanding the impact of schema changes

Tools Used

MySQL Community Server

MySQL Workbench

Files Included

intern_training_db.sql – Database schema with constraints

task2_table_design.sql – Table creation, constraints, and ALTER commands

Key Concepts Covered

Data types: INT, VARCHAR, DATE, TIMESTAMP

Constraints: PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT

Schema evolution: ADD, RENAME, DROP columns

Data integrity and validation rules

How to Run

Open MySQL Workbench

Create or select a database connection

Open the SQL script using File → Open SQL Script

Execute the script to create and modify the table

Learning Outcome

By completing this task, the intern understands how proper schema design and constraints help maintain accurate and reliable data in real-world database systems.
